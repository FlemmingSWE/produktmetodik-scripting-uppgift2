Koncept och Spelmekanik:
Applikationen är uppbyggd av temabaserade rum som testar spelarens säkerhetsmedvetenhet.

Progression: 
Varje avklarat rum ger en pusselbit. Samtliga pusselbitar krävs för att låsa upp slutdörren och klara spelet.

Livsystem: 
Spelaren startar med tre liv. Rätt beslut belönas med ett extra liv, 
medan felaktiga val straffas med ett förlorat liv. Om liven tar slut avbryts spelet.

Variation i utmaningar: 
Ett av rummen introducerar en "Hänga gubbe"-mekanik med en visuell progress bar, 
där spelaren ska identifiera säkerhetsbegrepp under press.

Pedagogiskt syfte (ADKAR)
Designen bygger på att stänga gapet mellan kunskap (Knowledge) och förmåga (Ability). 
Rummen är konstruerade så att användaren först presenteras för ett koncept (t.ex. nätfiske eller lösenordspolicy) 
och sedan omedelbart måste fatta ett aktivt beslut. Konsekvenssystemet med liv gör att rätt beteenden förstärks och kan appliceras i det dagliga arbetet.

Tekniska krav och Omfattning:
För att projektet ska anses godkänt ska följande tekniska grundkrav vara uppfyllda:

Miljö: 
Fullt fungerande terminalapplikation med tydligt flöde mellan rummen.

Datahantering: 
Möjlighet att spara och ladda spelarens framsteg via JSON.

Arkitektur: 
Koden ska vara modulärt uppbyggd i separata PS1-filer.

Stabilitet: 
Omfattande felhantering med Try/Catch i samtliga kritiska funktioner.

Arbetsmetodik:
Projektet drivs agilt med korta sprintar. Versionshantering sker via GitHub. Löpande dokumentation, 
insikter från daily stand-ups samt reflektioner från sprint reviews och retrospectives uppdateras kontinuerligt i denna README.
